# xy-cloud-skills 流水线约束

## STATUS.md 更新规则（必须执行）

每当任一技能（M1~M7）完成总线轨写入后，必须更新
`skills/shared/outputs/STATUS.md`：

1. 读取 STATUS.md；若不存在则创建，先写入标题行和
   图例 `# Pipeline Status` +
   `> ✅ = 最新  ⚠️ = 上游已更新建议重跑  📋 = 契约已就绪（待执行）  — = 未执行`
2. 找到或创建 `## tag: {tag}` 区块
3. 更新当前 M{N} 格：
   - M4：填入 `{当前时间戳} 📋`（契约已就绪）
   - 其余 M{N}：填入 `{当前时间戳} ✅`
   - 时间戳格式 YYYYMMDD-HHMM
4. 遍历该行 M1→M7：若某格时间戳 < 其左侧任一格的时间戳，标记 ⚠️；
   否则保持原标记（✅ / 📋 / —）
5. 写入文件

## 启动检查规则（必须执行）

每个技能启动时，先读取 STATUS.md 中当前 tag 行。
若本技能的上游技能（如 M3 的上游是 M1、M2）有 ⚠️ 标记，
必须在输出中显式提醒用户"上游产物已过期，建议从 M{X} 开始重跑"。

> 📋 不等于 ⚠️。M4 📋 表示 YAML 实现契约已就绪，M5/M6 可以正常读取。

## M4 契约模式说明

M4 不再生成自由文本任务书，而是输出 **YAML Implementation Contract**
（`shared/outputs/m4-契约/{tag}.yaml`），包含 create/modify 操作清单、
路径锚点、验证规则。代码执行 Agent 按契约逐条执行。

📋 标记仅用于 M4，表示"契约已生成，待执行"。

## M6 任务书模式说明

M6 沿用任务书模式：输出 `shared/outputs/m6-测试任务/{tag}.md`。
测试 Agent 读取此文件在目标项目生成测试代码。
