## 2. 四种测试基类

所有基类位于 `cn.shutan.platform.framework.test.core.ut`，按依赖范围递增：

### 2.1 BaseMockitoUnitTest — 纯 Mockito 单元测试

**用途**：纯逻辑测试，不启动 Spring 容器，不依赖数据库/Redis。适用于工具类、Controller 层、无数据库依赖的 Service 逻辑。

**注解**：`@ExtendWith(MockitoExtension.class)`

```java
public class XxxServiceImplTest extends BaseMockitoUnitTest {

    @InjectMocks
    private XxxServiceImpl xxxService;

    @Mock
    private XxxMapper xxxMapper;

    @Test
    public void testMethod_success() {
        when(xxxMapper.selectById(1L)).thenReturn(xxxDO);
        // 调用 + 断言
    }
}
```

### 2.2 BaseDbUnitTest — 数据库单元测试

**用途**：依赖 H2 内存数据库的 Service 层测试。启动 Spring 容器（不启动 Web 服务），每个方法后自动清理数据库。

**注解**：
- `@SpringBootTest(webEnvironment = NONE)`
- `@ActiveProfiles("unit-test")`
- `@Sql(scripts = "/sql/clean.sql", executionPhase = AFTER_TEST_METHOD)`

```java
@Import(XxxServiceImpl.class)
public class XxxServiceImplTest extends BaseDbUnitTest {

    @Resource
    private XxxServiceImpl xxxService;

    @Resource
    private XxxMapper xxxMapper;

    @MockBean
    private ExternalService externalService;

    @Test
    public void testMethod_success() {
        // 通过真实 Mapper 插入数据
        XxxDO do1 = randomPojo(XxxDO.class, o -> o.setField("value"));
        xxxMapper.insert(do1);
        // 调用真实服务
        List<XxxDO> list = xxxService.getList(param);
        // 断言
        assertEquals(1, list.size());
        assertPojoEquals(do1, list.get(0));
    }
}
```

### 2.3 BaseRedisUnitTest — Redis 单元测试

**用途**：依赖内嵌 Redis 的测试。使用 jedis-mock 启动内存 Redis Server。

**注解**：`@SpringBootTest(webEnvironment = NONE)` + `@ActiveProfiles("unit-test")`

### 2.4 BaseDbAndRedisUnitTest — 数据库 + Redis 单元测试

**用途**：同时依赖 H2 + 内嵌 Redis 的综合测试。

**注解**：`@SpringBootTest(webEnvironment = NONE)` + `@ActiveProfiles("unit-test")` + `@Sql(scripts = "/sql/clean.sql", executionPhase = AFTER_TEST_METHOD)`

### 基类选择决策

```
场景                                      → 选择
─────────────────────────────────────────────────────────
工具类/纯逻辑测试                          → BaseMockitoUnitTest
Controller 层（无 DB 依赖）                → BaseMockitoUnitTest
Service 层（依赖数据库）                   → BaseDbUnitTest
Service 层（依赖 Redis）                  → BaseRedisUnitTest
Service 层（依赖 DB + Redis）             → BaseDbAndRedisUnitTest
```

## 3. 测试工具类

### 3.1 RandomUtils — 随机数据生成

位于 `cn.shutan.platform.framework.test.core.util.RandomUtils`，底层使用 Podam + Hutool。

**核心方法**：

| 方法 | 返回 | 说明 |
|------|------|------|
| `randomString()` | String | 10 位随机字符串 |
| `randomLongId()` | Long | 0 ~ Long.MAX_VALUE |
| `randomInteger()` | Integer | 0 ~ Integer.MAX_VALUE |
| `randomLocalDateTime()` | LocalDateTime | 30 天内随机日期，纳秒置零 |
| `randomCommonStatus()` | Integer | 从 CommonStatusEnum 随机取值（0 或 1） |
| `randomEmail()` | String | 随机邮箱 |
| `randomMobile()` | String | 以 13800138 开头的 11 位手机号 |
| `randomURL()` | String | 以 `https://www.shutan.cn/` 开头的 URL |
| `randomPojo(Class, Consumer...)` | T | 生成随机 POJO，支持后置回调覆盖字段 |
| `randomPojoList(Class, Consumer...)` | List | 1~5 个随机 POJO 的 List |
| `randomPojoList(Class, int, Consumer...)` | List | 指定数量的随机 POJO List |

**使用示例**：

```java
// 生成随机 POJO
XxxDO do1 = randomPojo(XxxDO.class);

// 生成并覆盖特定字段
XxxDO do2 = randomPojo(XxxDO.class, o -> o.setName("test").setStatus(0));

// 批量生成
List<XxxDO> list = randomPojoList(XxxDO.class, 3,
    o -> o.setStatus(1));
```

**智能字段适配**（Podam 自定义类型制造商）：
- 字段名 `status` → 生成 0 或 1
- 字段名以 `type`/`status`/`category`/`scope`/`result` 结尾 → 0~127 范围
- 字段名 `deleted` → 固定 false
- Boolean 其他字段 → 随机 true/false

> ⚠️ **状态字段类型一致性提醒**: `randomCommonStatus()` 返回 `Integer` 类型（取值为 `CommonStatusEnum` 的 0 或 1），`randomPojo()` 对 status 字段默认生成 0 或 1（也是 `Integer`）。如果被测 DO 的 `status` 字段是 `String` 类型（如 eos 模块中部分实体），需在测试中显式覆盖 status 值，否则 Podam 可能跳过该字段或生成不匹配类型的数据。**建议**: system/infra 模块新增实体的 `status` 字段统一使用 `Integer` 类型，确保与测试工具默认行为一致。

### 3.2 AssertUtils — 增强断言

位于 `cn.shutan.platform.framework.test.core.util.AssertUtils`。

| 方法 | 说明 |
|------|------|
| `assertPojoEquals(expected, actual, ignoreFields...)` | 通过反射比对两个 POJO 的每个字段，支持忽略字段 |
| `assertServiceException(executable, errorCode, messageParams...)` | 断言抛出指定错误码的 ServiceException |

**使用示例**：

```java
// POJO 全字段比对
assertPojoEquals(expectedDO, actualDO);

// POJO 比对（忽略时间戳字段）
assertPojoEquals(expectedDO, actualDO, "createTime", "updateTime");

// 断言业务异常
assertServiceException(() -> xxxService.delete(999L),
    ErrorCodeConstants.DATA_NOT_EXISTS);
```

## 4. 测试配置文件

### 4.1 application-unit-test.yaml

每个有测试的业务模块需在 `src/test/resources/` 下提供。

**完整配置参考**（`platform-module-system`）：

```yaml
spring:
  main:
    lazy-initialization: true          # 延迟加载，加速测试
  banner-mode: OFF                     # 关闭 Banner
  datasource:
    url: jdbc:h2:mem:testdb;MODE=MYSQL;DATABASE_TO_LOWER=TRUE;CASE_INSENSITIVE_IDENTIFIERS=TRUE
    driver-class-name: org.h2.Driver
    username: root
    password: null
    schema: classpath:/sql/create_tables.sql  # 建表 SQL
  sql:
    init:
      mode: always                     # 每次初始化建表
      schema-locations: classpath:/sql/create_tables.sql
  h2:
    console:
      enable: true                     # H2 控制台（调试时可用）
  data:
    redis:
      host: 127.0.0.1
      port: 16379                      # jedis-mock 默认端口

# MyBatis-Plus 配置
mybatis-plus:
  configuration:
    map-underscore-to-camel-case: true
    cache-enabled: false
  global-config:
    db-config:
      id-type: AUTO                    # H2 使用自增主键
      logic-delete-value: 1
      logic-not-delete-value: 0

# 关闭外部依赖
lock4j:
  enabled: false
```

### 4.2 SQL 脚本

```
src/test/resources/sql/
├── create_tables.sql    # 实体对应表的 CREATE TABLE 语句（H2 语法）
└── clean.sql            # 清理语句：TRUNCATE 或 DELETE FROM 所有测试表
```

## 5. 测试代码规范

### 5.1 测试类命名

| 被测类型 | 测试类命名 | 示例 |
|---------|-----------|------|
| Service 实现 | `{XxxServiceImpl}Test` | `DeptServiceImplTest` |
| Controller | `{XxxController}Test` | `OAuth2OpenControllerTest` |
| Mapper | `{XxxMapper}Test` | — |
| 工具类 | `{XxxUtils}Test` | `CollectionUtilsTest` |
| Client | `{XxxClient}Test` | `AliyunSmsClientTest` |

### 5.2 测试方法命名

格式：`test{方法名}_{场景}`

| 场景 | 命名示例 |
|------|---------|
| 正常流程 | `testCreate_success` |
| 参数异常 | `testCreate_paramMissing` |
| 数据不存在 | `testDelete_notFound` |
| 状态冲突 | `testUpdate_statusConflict` |
| 权限不足 | `testQuery_noPermission` |

### 5.3 Mock 策略

| 场景 | 注解 | 基类 |
|------|------|------|
| 纯 Mockito（无 Spring） | `@Mock` + `@InjectMocks` | `BaseMockitoUnitTest` |
| Spring Boot 测试 | `@MockBean` + `@Resource` | `BaseDbUnitTest` |

**@Mock 模式**（Controller 测试典型案例）：

```java
public class XxxControllerTest extends BaseMockitoUnitTest {

    @InjectMocks
    private XxxController xxxController;

    @Mock
    private XxxService xxxService;

    @Test
    public void testGet_success() {
        when(xxxService.get(1L)).thenReturn(xxxDO);
        CommonResult<XxxRespVO> result = xxxController.get(1L);
        assertEquals(0, result.getCode());
    }
}
```

**@MockBean 模式**（Service 测试典型案例）：

```java
@Import(XxxServiceImpl.class)
public class XxxServiceImplTest extends BaseDbUnitTest {

    @Resource
    private XxxServiceImpl xxxService;

    @Resource
    private XxxMapper xxxMapper;        // 真实 Mapper

    @MockBean
    private ExternalService externalService;  // 外部依赖 Mock

    @Test
    public void testCreate_success() {
        // 准备
        XxxSaveReqVO reqVO = randomPojo(XxxSaveReqVO.class);
        when(externalService.validate(reqVO)).thenReturn(true);
        // 执行
        Long id = xxxService.create(reqVO);
        // 断言
        assertNotNull(id);
        XxxDO saved = xxxMapper.selectById(id);
        assertPojoEquals(reqVO, saved, "id", "createTime");
    }
}
```

### 5.4 测试数据准备

```java
// 方式一：随机 POJO + 覆盖特定字段（推荐）
XxxDO data = randomPojo(XxxDO.class,
    o -> o.setName("测试名称").setStatus(CommonStatusEnum.ENABLE.getStatus()));
xxxMapper.insert(data);

// 方式二：全字段手动设置（边界值测试）
XxxDO data = new XxxDO();
data.setId(1L);
data.setName("");  // 空字符串边界
data.setStatus(null);  // null 边界
xxxMapper.insert(data);

// 方式三：批量插入用于列表/分页测试
List<XxxDO> list = randomPojoList(XxxDO.class, 5,
    o -> o.setDeptId(deptId));
list.forEach(xxxMapper::insert);
```

### 5.5 断言模式

```java
// 1. 结果断言
assertEquals(expected, actual);
assertNotNull(actual);
assertTrue(condition);
assertIterableEquals(expectedList, actualList);

// 2. POJO 比对（推荐，替代逐个字段断言）
assertPojoEquals(expectedDO, actualDO);

// 3. POJO 比对（忽略时间戳等自动填充字段）
assertPojoEquals(expected, actual, "createTime", "updateTime", "creator");

// 4. 业务异常断言
assertServiceException(() -> xxxService.delete(999L), AUTH_LOGIN_BAD_CREDENTIALS);

// 5. Mock 交互验证
verify(xxxMapper).insert(any());
verify(externalService, never()).send(any());       // 验证未调用
verify(loginLogService, times(1)).createLoginLog(argThat(o -> ...));
```

## 6. 测试边界分析指南（M5 技能参考）

### 6.1 六种边界类型

| 边界类型 | 适用字段 | 示例 |
|---------|---------|------|
| **数值边界** | Integer, Long, BigDecimal, Double | 最小值、最大值、0、负数、超出范围 |
| **字符串边界** | String | 空串、超长、特殊字符、Null、仅空白 |
| **时间边界** | LocalDateTime, LocalDate, Date | 过去时间、未来时间、Null、格式异常 |
| **关联边界** | 外键 ID | 不存在、已删除、Null、跨租户 |
| **状态边界** | status 字段 | 已禁用、已删除、状态流转异常 |
| **业务规则** | 组合字段 | 唯一约束冲突、业务条件不满足、重复提交 |

### 6.2 边界值分析模板

```
字段名: {fieldName}
类型: {String/Integer/LocalDateTime/...}
边界条件:
  - [{场景描述}] → 输入: {value} → 预期: {expectedResult}
  - [{场景描述}] → 输入: {value} → 预期: {expectedResult}
```

### 6.3 常见边界场景

```java
// 数值边界
// - 创建时 name 超长（>50 字符）
// - 分页 pageNo ≤ 0
// - 排序 sort > Integer.MAX_VALUE

// 字符串边界
// - name = ""（空串，非 null）
// - name = null
// - code 包含特殊字符（空格、中文、SQL 注入尝试）

// 关联边界
// - deptId 指定的部门不存在
// - deptId = null（允许为空时）
// - 引用的数据已被逻辑删除

// 业务规则
// - 创建同名数据（唯一约束冲突）
// - 禁用状态下执行更新操作
// - 删除已被引用的数据
```
