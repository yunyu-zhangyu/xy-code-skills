## 1. 项目模块全景

| 模块目录 | 类型 | 说明 |
|---------|------|------|
| `platform-framework/` | 框架 | 基础库（common / mybatis / web / security / redis 等 starter） |
| `platform-gateway/` | 网关 | API 网关 |
| `platform-dependencies/` | BOM | 依赖版本管理 |
| `platform-module-system/` | 基础业务 | 用户/角色/菜单/部门等系统管理 |
| `platform-module-infra/` | 基础业务 | 配置/文件/日志等基础设施 |
| `platform-module-eos/` | **业务** | 能效运营系统（Energy Operation System） |
| `platform-module-eos-demand-response/` | **业务** | 需求响应模块 |

代码生成面向**业务模块**（标记为"业务"的模块）。

### 1.1 各模块功能一句话说明

| 模块 | 一句话功能 |
|------|----------|
| `platform-framework` | 所有模块的技术底座，提供 Web/Security/MyBatis/Redis/Protection 等 Starter |
| `platform-gateway` | 所有 API 请求的入口，负责路由转发和负载均衡 |
| `platform-dependencies` | 统一管理所有依赖的版本号，防止版本冲突 |
| `platform-module-system` | 用户、角色、菜单、部门、岗位等基础系统管理功能 |
| `platform-module-infra` | 文件存储、代码生成、参数配置、API 日志、定时任务等基础设施服务 |
| `platform-module-eos` | 能效运营核心业务，包含能源价格、数据项、指标等 |
| `platform-module-eos-demand-response` | 需求响应子模块，管理需求响应的策略和事件 |

## 2. 业务模块内部结构

每个业务模块分为 api 和 biz 两个子模块，以 `platform-module-eos` 为例：

```
xy-cloud/platform-module-eos/
├── platform-module-eos-api/                          # 轻量 JAR（Feign 接口 + 常量）
│   └── src/main/java/cn/shutan/platform/module/eos/
│       ├── api/                                      # Feign 客户端接口
│       ├── enums/                                    # ErrorCodeConstants, ApiConstants
│       └── dto/                                      # 响应 DTO（模块间通信用）
│
└── platform-module-eos-biz/                          # 可执行 JAR（业务实现）
    └── src/main/java/cn/shutan/platform/module/eos/
        ├── controller/
        │   ├── admin/{domain}/
        │   │   └── vo/                               # 管理后台 VO（SaveReqVO/PageReqVO/RespVO）
        │   └── app/{domain}/                         # APP 端 API（按需）
        ├── service/{domain}/                         # Service 接口 + Impl
        ├── dal/
        │   ├── dataobject/{domain}/                  # DO 实体
        │   └── mysql/{domain}/                       # MyBatis-Plus Mapper
        ├── config/                                   # 模块级配置
        ├── framework/                                # 模块级框架扩展
        └── job/                                      # XXL-Job 定时任务（按需）
```

### 2.1 各目录职责和典型文件

| 目录 | 职责 | 典型文件 |
|------|------|---------|
| `controller/admin/{domain}/` | 管理后台 API 端点 | `EnergyPriceController.java` |
| `controller/admin/{domain}/vo/` | 请求/响应 VO 对象 | `EnergyPriceSaveReqVO.java`, `EnergyPriceRespVO.java` |
| `controller/app/{domain}/` | APP 端 API（与 admin 路径隔离） | 按需创建 |
| `service/{domain}/` | 业务逻辑层 | `EnergyPriceService.java` + `EnergyPriceServiceImpl.java` |
| `dal/dataobject/{domain}/` | 数据库实体 | `EnergyPriceDO.java` |
| `dal/mysql/{domain}/` | MyBatis-Plus Mapper | `EnergyPriceMapper.java` |
| `config/` | 模块级 Spring 配置 | 数据源、线程池等配置 |
| `framework/` | 模块级框架扩展 | 拦截器、过滤器等自定义组件 |
| `job/` | XXL-Job 定时任务处理器 | 定时统计、清理任务 |
| `api/` (biz 模块中) | Feign 接口实现 | 实现 api 模块定义的 Feign 接口 |

### 2.2 请求全链路过滤器图

一个 HTTP 请求从 Gateway 入口到数据库的完整过滤器链顺序：

```
客户端请求
  │
  ▼
┌─────────────────────────────────────────────────────┐
│  Gateway 网关层（platform-gateway）                   │
│                                                       │
│  ① CorsFilter             ── CORS 跨域检查            │
│  ② AccessLogFilter        ── 网关访问日志              │
│  ③ GrayLoadBalancer       ── 灰度负载均衡（按 version）│
│  ④ TokenAuthenticationFilter ── 验证 Bearer token      │
│       │  └→ WebClient 调用 OAuth2TokenApi.checkAccessToken()  │
│       │     └→ Guava Cache 缓存 1 分钟                  │
│       └→ 序列化 LoginUser → 写入 login-user Header 透传  │
│                                                       │
│  路由到目标微服务（Nacos 服务发现）                     │
└────────────────────┬────────────────────────────────┘
                     │
                     ▼
┌─────────────────────────────────────────────────────┐
│  Web 层（platform-spring-boot-starter-web）           │
│                                                       │
│  ⑤ CacheRequestBodyFilter  ── 缓存 Request Body       │
│  ⑥ XssFilter              ── XSS 清洗                 │
│  ⑦ DemoFilter             ── 演示模式（禁止写操作）     │
│  ⑧ ApiAccessLogFilter     ── API 访问日志 + 参数脱敏   │
│                                                       │
│  ┌─ HandlerInterceptor ─┐                            │
│  │ ApiAccessLogInterceptor                            │
│  │  preHandle: 记录 HandlerMethod + 计时              │
│  │  afterCompletion: 记录耗时                         │
│  └──────────────────────┘                            │
└────────────────────┬────────────────────────────────┘
                     │
                     ▼
┌─────────────────────────────────────────────────────┐
│  Controller 层                                        │
│                                                       │
│  ⑨ @Validated 参数校验                                │
│  ⑩ @PreAuthorize 权限检查（AOP）                       │
│     └→ @ss.hasPermission() → PermissionApi (Feign)    │
│  ⑪ 调用 Service 方法                                  │
└────────────────────┬────────────────────────────────┘
                     │
                     ▼
┌─────────────────────────────────────────────────────┐
│  Service 层                                           │
│                                                       │
│  ⑫ @Transactional 事务管理                            │
│  ⑬ 业务逻辑校验                                        │
│  ⑭ 调用 Mapper 方法                                   │
└────────────────────┬────────────────────────────────┘
                     │
                     ▼
┌─────────────────────────────────────────────────────┐
│  MyBatis-Plus 层                                      │
│                                                       │
│  ⑮ TenantDatabaseInterceptor   ── 自动拼接 tenant_id  │
│  ⑯ DataPermissionInterceptor   ── 数据权限 WHERE 条件  │
│  ⑰ Mapper 执行 SQL → MySQL                            │
└─────────────────────────────────────────────────────┘
                     │
                     ▼
┌─────────────────────────────────────────────────────┐
│  响应返回                                             │
│                                                       │
│  ⑱ GlobalResponseBodyHandler  ── 记录 CommonResult    │
│  ⑲ GlobalExceptionHandler    ── 异常时统一包装错误响应  │
│  ⑳ 响应原路返回客户端                                  │
└─────────────────────────────────────────────────────┘
```

> **调试提示**: 遇到请求异常时，按上图标号顺序逐层排查。常见断点位置：④（token 过期）、⑩（权限不足）、⑮（租户 SQL 未拼接/静默失败）、⑲（异常被 GlobalExceptionHandler 包装）。

## 3. 生成文件 → 项目目录映射

### 3.1 变量说明

- `${module}` — 模块名小写，如 `eos`、`infra`、`system`
- `${domain}` — 领域包名，如 `energyprice`、`dataitem`
- `${entity}` — 实体名 PascalCase，如 `EnergyPrice`、`DataItem`
- `${tag}` — 需求标识，如 `price`、`data-item`

### 3.2 标准映射

M4 生成文件输出到 `D:\ZHYwork\Skills\技能包\shared\outputs\generated-code\{tag}\`，复制时按以下映射放入项目：

|---|---------|---------|----------------------------------------------------------------|
| 1 | `${entity}Controller.java` | `${module}-biz` | `controller/admin/${domain}/` |
| 2 | `${entity}Service.java` | `${module}-biz` | `service/${domain}/` |
| 3 | `${entity}ServiceImpl.java` | `${module}-biz` | `service/${domain}/` |
| 4 | `${entity}Mapper.java` | `${module}-biz` | `dal/mysql/${domain}/` |

## 5. 操作流程

1. **M4 生成** — 在 `D:\ZHYwork\Skills\技能包\shared\outputs\generated-code\{tag}\` 产出 10 个文件（9 Java + 1 XML）
2. **定位目标** — 根据 `${module}` 查**模块路径速查表**找到 biz 根目录
3. **建包** — 在 biz 根目录下创建 `${domain}` 对应的子目录（如果还不存在）
4. **复制 Java 源文件** — 将 9 个 `.java` 文件放入对应 `src/main/java/...` 目录
5. **复制资源文件** — 将 `${entity}Mapper.xml` 放入 `${module}-biz/src/main/resources/mapper/` 目录
6. **检查 pom.xml** — 确认目标模块的 `pom.xml` 包含所需 starter
7. **检查 import** — 验证所有 import 路径正确（IDE 会自动高亮错误）
8. **编译** — `mvn compile -pl platform-module-${module}/platform-module-${module}-biz -am` 验证

### 5.1 常见错误和排查

| 错误 | 原因 | 排查方法 |
|------|------|---------|
| `package does not exist` | 源文件放错目录或包名不匹配 | 检查文件的 `package` 声明与物理路径是否一致 |
| `cannot find symbol` | 缺少 starter 依赖 | 检查 pom.xml 的 dependency |
| `BindingException` | Mapper XML 缺失 | 确认 XML 在 `resources/mapper/` 下 |
| 编译通过但运行时 404 | Controller 未注册到 Spring | 检查 `@RestController` 注解和包扫描路径 |

### 7.1 功能总览

| 功能 | 说明 | 对其他模块的价值 |
|------|------|----------------|

| 定时任务 | XXL-Job 集成 + 内置清理任务 | 各模块统一管理 |
| Redis 监控 | Redis 服务器状态查询 | 运维监控 |
| WebSocket | 多节点消息推送 | 即时通讯参考实现 |

### 7.2 Feign API 接口

**ConfigApi — 参数配置**
```java
CommonResult<String> getConfigValueByKey(@RequestParam("key") String key);
```

**FileApi — 文件操作**
```java
String createFile(String name, String path, byte[] content);  // 返回文件 URL
byte[] getFileContent(Long configId, String path);
void deleteFileById(Long fileId);
```

**WebSocketSenderApi — WebSocket 推送**
```java
void send(Integer userType, Long userId, String messageType, String messageContent);
void sendObject(Integer userType, Long userId, String messageType, Object messageContent);
```

**ApiAccessLogApi / ApiErrorLogApi — 日志**
```java
CommonResult<Boolean> createApiAccessLog(ApiAccessLogCreateReqDTO createDTO);
CommonResult<Boolean> createApiErrorLog(ApiErrorLogCreateReqDTO createDTO);
```

### 7.3 常量参考

- 服务名: `ApiConstants.NAME` = `infra-server`

- 错误码段: 1-001-xxx-xxx

## 8. 框架层 Starter 速览

`platform-framework` 下的所有 starter 及其一句话用途：

| Starter | 一句话用途 |
|---------|----------|
| `platform-common` | 核心工具类和通用模型（Result、Page、ErrorCode） |
| `platform-spring-boot-starter-web` | MVC + Tomcat + Jackson + Swagger 整合 |
| `platform-spring-boot-starter-security` | Spring Security 集成 + @PreAuthorize 权限 |
| `platform-spring-boot-starter-mybatis` | MyBatis-Plus + 动态数据源 + 分页 |
| `platform-spring-boot-starter-redis` | Redis 缓存 + Redisson 分布式锁 |
| `platform-spring-boot-starter-protection` | 接口防护：@Idempotent 幂等 + @RateLimiter 限流 |
| `platform-spring-boot-starter-biz-tenant` | 多租户：TenantDatabaseInterceptor 自动 SQL 拼接 |

## 9. 模块间调用关系图

文字版架构图，展示 system/infra/eos 之间的 Feign 调用关系：

```
                      ┌────────────────────┐
                      │  platform-gateway   │
                      │  (API 网关/路由)     │
                      └──────┬─────────────┘
                             │
          ┌──────────────────┼──────────────────┐
          │                  │                  │
     ┌────▼────┐       ┌────▼────┐       ┌────▼────┐
     │ system  │       │  infra  │       │   eos   │
     │ (用户/  │       │ (文件/  │       │ (能效/  │
     │  角色/  │       │  配置/  │       │  需求)  │
     │  权限)  │       │  日志)  │       │         │
     └────┬────┘       └────┬────┘       └────┬────┘
          │                 │                 │
          │                 │                 │
          └──────────┬──────┘                 │
                     │                        │
                ┌────▼────┐                   │
                │  infra  │◄──────────────────┘
                │  Server │   eos 调用 infra 的:
                │         │   - ConfigApi (读取配置)
                │         │   - FileApi (文件上传)
                │         │   - ApiAccessLogApi (日志上报)
                └─────────┘
```

**调用关系说明**：

- `system` ↔ `infra`: 双向 Feign 调用。system 需要 infra 的文件和配置服务，infra 需要 system 的用户和权限信息。
- `eos` → `infra`: eos 作为业务模块，依赖 infra 的文件存储、参数配置、API 日志等基础服务。eos 不直接调用 system，通过 infra 间接获取用户/权限信息。
- `eos` ↔ `eos-demand-response`: 同一业务域内，demand-response 子模块通过 Feign 调用 eos 主模块的数据接口。

> 所有 Feign 调用均通过服务名发现（Nacos），不硬编码 IP 和端口。
