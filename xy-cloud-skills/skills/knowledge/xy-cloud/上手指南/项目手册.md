# xy-cloud 项目手册

> M0 基于项目扫描结果生成 | 2026-06-12

## 这个项目是做什么的？

xy-cloud 是一个**能效运营管理平台**（Energy Operation System），基于 Spring Cloud 微服务架构，面向能源管理场景提供 SaaS 服务。技术栈以 Spring Boot 2.7.18 + Spring Cloud Alibaba + MyBatis-Plus + Redis 为主。

**技术栈摘要：**

| 维度 | 技术 |
|------|------|
| 语言 | Java 8 |
| 框架 | Spring Boot 2.7.18, Spring Cloud Alibaba |
| ORM | MyBatis-Plus 3.5.x |
| 数据库 | MySQL (主), 支持 PostgreSQL/Oracle |
| 缓存 | Redis (Spring Cache + Redisson) |
| 消息 | Redis Stream (默认), RabbitMQ/RocketMQ/Kafka 可选 |
| 注册中心 | Nacos |
| 网关 | Spring Cloud Gateway |
| 定时任务 | XXL-Job |

## 开发一个新功能的完整流程

这个项目采用标准的 Controller → Service → Mapper → DO 四层架构。下面以 Config（参数配置）功能为完整例子，逐层说明。

### Step 1: 创建 DO（数据对象）

文件：`platform-module-infra/platform-module-infra-biz/src/main/java/cn/shutan/platform/module/infra/dal/dataobject/config/ConfigDO.java`

- 继承 `BaseDO`（自动获得 createTime、updateTime、creator、updater、deleted 字段）
- 使用 `@TableName` 指定表名
- 使用 `@TableId` 标记主键（项目用 MySQL 时 @KeySequence 可不写但通常会保留）

**框架自动做了：** BaseDO 的 5 个字段由 MyBatis-Plus 自动填充，不需要你在代码中 set。

### Step 2: 创建 Mapper（数据访问）

文件：`platform-module-infra/platform-module-infra-biz/src/main/java/cn/shutan/platform/module/infra/dal/mysql/config/ConfigMapper.java`

- 继承 `BaseMapperX<ConfigDO>`（自动获得 CRUD + 批量操作）
- 使用 `LambdaQueryWrapperX` 构建查询条件（likeIfPresent、eqIfPresent、betweenIfPresent 等方法）
- 返回 `PageResult<ConfigDO>` 配合分页

**框架自动做了：** 基础的 insert、updateById、deleteById、selectById 都在 BaseMapperX 中，不需要写任何 SQL。

### Step 3: 创建 SaveReqVO（创建/更新请求体）

文件：`platform-module-infra/platform-module-infra-biz/src/main/java/cn/shutan/platform/module/infra/controller/admin/config/vo/ConfigSaveReqVO.java`

- 使用 `@Schema` 做 API 文档描述
- 使用 `@NotEmpty / @NotBlank / @NotNull / @Size` 做参数校验

### Step 4: 创建 RespVO（响应体）

文件：`platform-module-infra/platform-module-infra-biz/src/main/java/cn/shutan/platform/module/infra/controller/admin/config/vo/ConfigRespVO.java`

- 如果功能需要 Excel 导出，加 `@ExcelProperty` + `@ExcelIgnoreUnannotated`
- 字典字段使用 `@DictFormat` + `DictConvert` 自动转换

### Step 5: 创建 Service 接口 + 实现

接口文件：`platform-module-infra/platform-module-infra-biz/src/main/java/cn/shutan/platform/module/infra/service/config/ConfigService.java`
实现文件：`platform-module-infra/platform-module-infra-biz/src/main/java/cn/shutan/platform/module/infra/service/config/ConfigServiceImpl.java`

- Service 只对 Controller 层暴露，包含 create / update / delete / get / getPage 标准方法
- 业务校验在 Service 中通过 `ServiceExceptionUtil.exception(ErrorCode)` 抛出

### Step 6: 创建 Convert（DO ↔ VO 转换）

文件：`platform-module-infra/platform-module-infra-biz/src/main/java/cn/shutan/platform/module/infra/convert/config/ConfigConvert.java`

- 使用 MapStruct 编译期生成实现
- Controller 通过 Convert 将 DO 转为 VO 返回给前端
- 示例：`@Mapping(source = "configKey", target = "key")` 做字段名映射

### Step 7: 创建 Controller（API 入口）

文件：`platform-module-infra/platform-module-infra-biz/src/main/java/cn/shutan/platform/module/infra/controller/admin/config/ConfigController.java`

- RESTful 风格，请求路径 `/{module}/{resource}`
- 统一返回 `CommonResult<T>`（框架自动包装）
- 权限控制 `@PreAuthorize("@ss.hasPermission('{module}:{resource}:{action}')")`

### Step 8: 注册路由

在对应模块的 `pom.xml` 确认已引入 web / security / mybatis 这三个必须的 starter：

```xml
<dependency>
    <groupId>cn.shutan.platform</groupId>
    <artifactId>platform-spring-boot-starter-web</artifactId>
</dependency>
<dependency>
    <groupId>cn.shutan.platform</groupId>
    <artifactId>platform-spring-boot-starter-security</artifactId>
</dependency>
<dependency>
    <groupId>cn.shutan.platform</groupId>
    <artifactId>platform-spring-boot-starter-mybatis</artifactId>
</dependency>
```

完成后用 `mvn compile -pl platform-module-{name}/platform-module-{name}-biz -am` 编译验证。

### 请求链路示例

以"创建参数配置"为例，请求从浏览器到数据库的完整链路：

1. `POST /infra/config/create`（带 JSON body）
2. `ConfigController.createConfig()` — 接收请求，调用 Service
3. `ConfigServiceImpl.createConfig()` — 从 `ConfigSaveReqVO` 转换 DO，调用 Mapper 插入
4. `ConfigMapper.insert()` — BaseMapperX 提供的通用方法
5. MySQL 写入 `infra_config` 表
6. 返回 `CommonResult<Long>`（新建记录的 ID）
