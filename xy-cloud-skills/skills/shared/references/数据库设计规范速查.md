# 数据库设计规范速查

## 1. 表命名规范

**格式**: `{module}_{entity}`（全小写 snake_case）

| 模块前缀 | 对应模块 |
|---------|---------|
| `system_` | platform-module-system |
| `infra_` | platform-module-infra |
| `eos_` | platform-module-eos |
| `demand_` | platform-module-eos-demand-response |

**规则**:
- 全小写，下划线分隔
- 实体名单数：`dept`, `role`, `menu`, `config`
- 避免关键字用复数：`system_users`
- 关联表用实体名组合：`system_role_menu`, `system_user_role`

**@TableName**:
```java
@TableName("system_dept")
// ⚠️ 有 JSON 字段时必须开启 autoResultMap = true，否则字段查询返回 null（不报错！）
@TableName(value = "system_users", autoResultMap = true)
```

## 2. DO 继承体系

### 2.1 BaseDO（所有 DO 基类）
`cn.shutan.platform.framework.mybatis.core.dataobject.BaseDO`

| 字段 | 类型 | 注解 | 填充策略 |
|------|------|------|---------|
| id | Long | `@TableId` | `NONE`（智能适配，MySQL → AUTO 自增） |
| createTime | LocalDateTime | `@TableField(fill = INSERT)` | 插入时自动填充 |
| updateTime | LocalDateTime | `@TableField(fill = INSERT_UPDATE)` | 更新时自动填充 |
| creator | String | `@TableField(fill = INSERT)` | 插入时自动填充 |
| updater | String | `@TableField(fill = INSERT_UPDATE)` | 更新时自动填充 |
| deleted | Boolean | `@TableLogic` | 逻辑删除标记 |

**额外实现**: `Serializable` + `TransPojo`（Easy-Trans 字典翻译）

### 2.2 TenantBaseDO（多租户扩展）
`cn.shutan.platform.framework.tenant.core.db.TenantBaseDO`

| 字段 | 类型 | 说明 |
|------|------|------|
| tenantId | Long | 租户 ID |

DDL: `"tenant_id" bigint not null default '0'`

### 2.3 继承选择原则

| 场景 | 继承 | 判断标准 |
|------|------|---------|
| 系统级表（字典/菜单/岗位/配置/通知/日志等） | `BaseDO` | 全局共享，无需租户隔离 |
| 租户级表（部门/用户/角色等） | `TenantBaseDO` | 数据按租户隔离 |

> ⚠️ **选错的后果**：租户表继承 `BaseDO` → `TenantDatabaseInterceptor` 不会自动拼接 `tenant_id` 过滤 → **所有租户数据互相可见**。无报错、无异常、日志无痕迹。

## 5. 自动填充机制

**实现类**: `cn.shutan.platform.framework.mybatis.core.handler.DefaultDBFieldHandler`
**机制**: 实现 `MetaObjectHandler`，仅 `BaseDO` 实例且字段值为 **null** 时填充。

**insert 填充**:
| 字段 | 填充值 |
|------|--------|
| createTime | `LocalDateTime.now()` |
| updateTime | `LocalDateTime.now()` |
| creator | `LoginUser.getId().toString()` |
| updater | `LoginUser.getId().toString()` |

**update 填充**:
| 字段 | 填充值 |
|------|--------|
| updateTime | `LocalDateTime.now()` |
| updater | `LoginUser.getId().toString()` |

**@TableField 填充策略**:
| 策略 | 说明 |
|------|------|
| `FieldFill.INSERT` | 仅插入时自动填充 |
| `FieldFill.INSERT_UPDATE` | 插入和更新时自动填充 |
| `FieldFill.UPDATE` | 仅更新时自动填充 |
| `FieldFill.DEFAULT` | 不自动填充 |

## 6. @TableField 注解模式

| 用途 | 示例 | 说明 |
|------|------|------|
| 自动填充 | `@TableField(fill = FieldFill.INSERT)` | 配合 MetaObjectHandler |
| JSON 处理器 | `@TableField(typeHandler = JacksonTypeHandler.class)` | List/Set/Map 序列化为 JSON |
| 非数据库字段 | `@TableField(exist = false)` | Java 内存字段 |
| 强制更新 | `@TableField(updateStrategy = FieldStrategy.ALWAYS)` | null 值也执行 UPDATE |
| 加密处理器 | `@TableField(typeHandler = EncryptTypeHandler.class)` | 敏感字段透明加解密 |
| 指定 JDBC 类型 | `@TableField(jdbcType = JdbcType.VARCHAR)` | 解决类型推断问题 |

**JacksonTypeHandler 场景**（必须开启 `autoResultMap = true`）:
| DO 类 | 字段 | Java 类型 |
|-------|------|-----------|
| AdminUserDO | postIds | `Set<Long>` |
| TenantPackageDO | menuIds | `Set<Long>` |
| OAuth2AccessTokenDO | scopes | `List<String>` |
| SmsTemplateDO | params | `List<String>` |
| SmsLogDO | templateParams | `Map<String, Object>` |
| RoleDO | dataScopeDeptIds | `Set<Long>` |
| FileConfigDO | config | `Map<String, Object>` |

## 7. 逻辑删除

```java
@TableLogic
private Boolean deleted;
```

```sql
"deleted" bit NOT NULL DEFAULT FALSE
```

```yaml
mybatis-plus:
  global-config:
    db-config:
      logic-delete-value: 1        # 已删除
      logic-not-delete-value: 0     # 未删除
```

**查询自动追加 `WHERE deleted = 0`**，自定义 SQL 需显式加条件：
```sql
AND u.deleted = 0
```

## 8. 多租户

**字段**: `tenant_id bigint NOT NULL DEFAULT '0'`

**自动过滤**: `TenantDatabaseInterceptor` 实现 `TenantLineHandler`，自动拼接 `AND tenant_id = ?`。

**忽略租户过滤的表**:
```yaml
platform:
  tenant:
    ignore-tables:
      - system_tenant
      - system_dict_type
      - system_dict_data
      - infra_config
      # ... 其他系统级配置表
```

**编程式忽略**:
```java
// 注解方式
@TenantIgnore
public void someMethod() { ... }

// 编程方式
TenantUtils.executeIgnore(() -> {
    return someService.listAll();
});
```

## 10. 特殊规则

| 规则 | 说明 |
|------|------|
| 乐观锁 | **不使用** `@Version`，用悲观锁或分布式锁 |
| 索引定义 | **DDL 语句**，不使用 `@TableIndex`：`CREATE INDEX idx_xxx ON table(col)` |
| 枚举字段 | **`@InEnum`** 校验，数据库存 int，不启用枚举类型处理器 |
| 加密存储 | `@TableField(typeHandler = EncryptTypeHandler.class)` 透明加解密 |
| eos 多租户 | eos 表含 tenant_id，但 DO 继承 BaseDO，XML Mapper 中手动拼条件 |
| system/infra 状态码 | system/infra 模块状态字段**必须**使用 `Integer/tinyint(3)`（0=启用 1=停用），**禁止使用 varchar** |
| eos 状态码 | eos 模块使用 varchar(8) 字符串编码（如 '01'、'0201'），非 tinyint 枚举 |

## 11. Mapper 查询模式

基于实际项目代码整理的 MyBatis-Plus Mapper 层通用查询模式。
基类：`BaseMapperX<T>` | 查询包装器：`LambdaQueryWrapperX` / `MPJLambdaWrapperX`

### 11.1 单表分页查询

每个实体 Mapper 通过 default 方法提供分页查询，使用 `BaseMapperX.selectPage(PageParam, Wrapper)`。

```java
@Mapper
public interface DataItemMapper extends BaseMapperX<DataItemDO> {
    default PageResult<DataItemDO> selectPage(DataItemPageReqVO reqVO) {
        return selectPage(reqVO, new LambdaQueryWrapperX<DataItemDO>()
                .eqIfPresent(DataItemDO::getField, reqVO.getField())
                .likeIfPresent(DataItemDO::getName, reqVO.getName())
                .betweenIfPresent(DataItemDO::getCreateTime, reqVO.getCreateTime())
                .orderByDesc(DataItemDO::getId));
    }
}
```

#### 条件方法速查

| 方法 | 说明 | 空值处理 |
|------|------|---------|
| `eqIfPresent(column, val)` | 等于 | val 为 null 时跳过 |
| `neIfPresent(column, val)` | 不等于 | val 为 null 时跳过 |
| `likeIfPresent(column, val)` | 模糊匹配 | val 为 null/空串 时跳过 |
| `gtIfPresent(column, val)` | 大于 | val 为 null 时跳过 |
| `geIfPresent(column, val)` | 大于等于 | val 为 null 时跳过 |
| `ltIfPresent(column, val)` | 小于 | val 为 null 时跳过 |
| `leIfPresent(column, val)` | 小于等于 | val 为 null 时跳过 |
| `betweenIfPresent(column, val1, val2)` | 范围 | 任一为 null 时跳过 |
| `betweenIfPresent(column, val[])` | 范围（数组） | 数组为 null/空时跳过 |
| `inIfPresent(column, values)` | IN 查询 | 集合为空时跳过 |
| `orderByDesc(column)` | 降序排序 | — |
| `orderByAsc(column)` | 升序排序 | — |

### 11.2 单表列表查询

用于下拉选择、简单列表查询，通常在 Mapper 中定义 default 方法：

```java
default List<DataItemDO> selectList(DataItemListReqVO reqVO) {
    return selectList(new LambdaQueryWrapperX<DataItemDO>()
            .eqIfPresent(DataItemDO::getStatus, reqVO.getStatus()));
}

// 或直接使用继承的现成方法
xxMapper.selectList(EntityDO::getField, value);     // 按字段查询列表
xxMapper.selectOne(EntityDO::getField, value);      // 查询单个
xxMapper.selectCount(EntityDO::getField, value);    // 计数
```

### 11.3 多表 Join 分页查询

使用 `MPJLambdaWrapperX`（mybatis-plus-join 增强），Mapper 需继承 `BaseMapperX<T>`（已继承 `MPJBaseMapper`）。

```java
default PageResult<DataItemRespVO> selectJoinPage(DataItemPageReqVO reqVO) {
    return selectJoinPage(reqVO, DataItemRespVO.class,
            new MPJLambdaWrapperX<DataItemDO>()
                .selectAll(DataItemDO.class)
                .selectAs(OtherDO::getName, DataItemRespVO::getOtherName)
                .leftJoin(OtherDO.class, OtherDO::getId, DataItemDO::getOtherId)
                .eqIfPresent(DataItemDO::getStatus, reqVO.getStatus()));
}
```

| 方法 | 用法 |
|------|------|
| `leftJoin(Class, on1, on2)` | LEFT JOIN |
| `innerJoin(Class, on1, on2)` | INNER JOIN |
| `selectAll(DO.class)` | 查询主表所有字段 |
| `selectAs(DO::getField, VO::getField)` | 查询副表字段并别名映射 |

### 11.4 批量操作

所有批量方法由 `BaseMapperX` 提供，直接调用：

```java
// 批量插入
xxMapper.insertBatch(entityList);

// 批量更新
xxMapper.updateBatch(entityList);

// 按字段批量查询
List<EntityDO> list = xxMapper.selectList(EntityDO::getStatus, 0);

// 按字段批量删除
xxMapper.delete(EntityDO::getField, value);
xxMapper.delete(EntityDO::getField, valueList);
```

### 11.5 Service 层调用示例

```java
@Service
public class EntityServiceImpl implements EntityService {

    @Resource
    private EntityMapper entityMapper;

    @Override
    public PageResult<EntityDO> getEntityPage(EntityPageReqVO pageReqVO) {
        return entityMapper.selectPage(pageReqVO);
    }

    @Override
    public List<EntityDO> getEntityList(EntityListReqVO reqVO) {
        return entityMapper.selectList(reqVO);
    }
}
```

### 11.6 Mapper XML 文件规范

MyBatis-Plus 单表 CRUD 不需要 XML。当需要多表关联或复杂 SQL 时，在 `src/main/resources/mapper/` 下创建 `{Entity}Mapper.xml`，具体规范见 `代码规范速查.md` 第 9 节。

XML 中仅编写 `<select>` 查询（不写 insert/update/delete），使用 `resultType` 全限定类名，动态 SQL 使用 `<where>` + `<if>` 标签，所有查询需显式加 `AND deleted = 0`。

## 12. 存储过程约定

项目中 eos 模块使用存储过程处理复杂数据校验和设备创建逻辑。

**命名**: `{模块}_{功能}`，如 `check_energy_data`、`check_org_data`、`create_ems_device`

| 存储过程 | 说明 |
|---------|------|
| `check_energy_data(p_org_str, p_start, p_end)` | 检查指定组织在日期范围内的能量数据完整性 |
| `check_org_data(org_id, return_mode)` | 检查组织的基础数据配置完整性 |
| `create_ems_device(org_id, parent_id, device_start_id, ...)` | 创建 EMS 设备及其关联的测量点 |

**约定**:
- 输入参数使用 `IN` 前缀
- 不使用 `OUT` 参数，通过 SELECT 结果集返回数据
- 异常处理使用 `DECLARE CONTINUE HANDLER`
