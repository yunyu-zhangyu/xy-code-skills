## 1. 四种测试基类

| 基类 | 注解 | 用途 |
|------|------|------|
| `BaseMockitoUnitTest` | `@ExtendWith(MockitoExtension.class)` | 纯 Mockito，无 Spring 容器 |
| `BaseDbUnitTest` | `@SpringBootTest(web=NONE)` + `@ActiveProfiles("unit-test")` + `@Sql(clean.sql)` | H2 内存数据库 |
| `BaseRedisUnitTest` | `@SpringBootTest(web=NONE)` + `@ActiveProfiles("unit-test")` | 内嵌 Redis |
| `BaseDbAndRedisUnitTest` | 两者组合 | H2 + Redis |

基类位于 `cn.shutan.platform.framework.test.core.ut`。

## 2. 测试工具类

### RandomUtils（`cn.shutan.platform.framework.test.core.util.RandomUtils`）

| 方法 | 说明 |
|------|------|
| `randomPojo(Class, Consumer...)` | 生成随机 POJO，后置回调覆盖字段 |
| `randomPojoList(Class, int, Consumer...)` | 批量随机 POJO |
| `randomString()` | 10 位随机字符串 |
| `randomLongId()` / `randomInteger()` | 随机 ID / 整数 |
| `randomCommonStatus()` | 0 或 1 |
| `randomEmail()` / `randomMobile()` / `randomURL()` | 随机邮箱/手机/URL |

### AssertUtils（`cn.shutan.platform.framework.test.core.util.AssertUtils`）

| 方法 | 说明 |
|------|------|
| `assertPojoEquals(expected, actual, ignoreFields...)` | 反射比对 POJO 所有字段 |
| `assertServiceException(executable, errorCode, params...)` | 断言业务异常抛出 |

## 3. Mock 策略

| 场景 | 注解 | 基类 |
|------|------|------|
| 纯 Mockito | `@InjectMocks` (类) + `@Mock` (字段) | `BaseMockitoUnitTest` |
| Spring Boot | `@Resource` (字段) + `@MockBean` (外部依赖) | `BaseDbUnitTest` + `@Import(XxxServiceImpl.class)` |

## 4. 测试配置

**Profile**：`unit-test`（通过 `@ActiveProfiles("unit-test")` 激活）

**关键配置**（`src/test/resources/application-unit-test.yaml`）：
```yaml
spring:
  datasource:
    url: jdbc:h2:mem:testdb;MODE=MYSQL;DATABASE_TO_LOWER=TRUE;CASE_INSENSITIVE_IDENTIFIERS=TRUE
    driver-class-name: org.h2.Driver
    schema: classpath:/sql/create_tables.sql
  data:
    redis:
      host: 127.0.0.1
      port: 16379
mybatis-plus:
  global-config:
    db-config:
      id-type: AUTO
```

**SQL 脚本**：
- `src/test/resources/sql/create_tables.sql` — 建表
- `src/test/resources/sql/clean.sql` — 清理（`@Sql(scripts = "/sql/clean.sql", executionPhase = AFTER_TEST_METHOD)`）

## 5. 编码规范

### 命名
- 测试类：`{被测类名}Test.java`（如 `DeptServiceImplTest`）
- 测试方法：`test{方法名}_{场景}`（如 `testCreate_success`、`testDelete_notFound`）

### 测试数据准备
```java
// 推荐：随机 POJO + 覆盖特定字段
XxxDO data = randomPojo(XxxDO.class, o -> o.setName("test"));
xxxMapper.insert(data);
```

### 断言模式
```java
assertEquals(expected, actual);
assertPojoEquals(expectedDO, actualDO);              // POJO 全字段比对
assertPojoEquals(expectedDO, actualDO, "createTime"); // 忽略自动填充字段
assertServiceException(() -> svc.delete(999L), ERROR_CODE); // 业务异常
verify(xxxMapper).insert(any());
verify(externalService, never()).send(any());
```

## 7. 边界分析类型（M5 参考）

| 边界类型 | 适用字段 | 示例场景 | 典型优先级 |
|---------|---------|---------|-----------|
| 数值边界 | Integer/Long/BigDecimal | 最小值、最大值、0、负数 | P0: @Min/@Max越界 / P1: 精度溢出 / P2: 震荡值 |
| 字符串边界 | String | 空串、超长、Null、特殊字符 | P0: null/超长/SQL注入 / P1: 格式错误 / P2: Unicode |
| 时间边界 | LocalDateTime/Date | 过去、未来、Null | P0: 必填字段null / P1: 时间先后颠倒 / P2: 时区 |
| 关联边界 | 外键 ID | 不存在、已删除、Null | P0: 不存在ID / P1: 已删除/跨租户 / P2: 自我引用 |
| 状态边界 | status | 已禁用、已删除、非法流转 | P0: 非法枚举值 / P1: 非法流转 / P2: 重复操作 |
| 业务规则 | 组合字段 | 唯一约束冲突、重复提交 | P0: 唯一冲突 / P1: 业务约束 / P2: 组合约束 |

### 优先级定义
- **P0（严重）**：崩溃/数据损坏/安全隐患——必须自动化测试覆盖
- **P1（中等）**：功能逻辑异常——需要测试覆盖
- **P2（轻微）**：边缘情况/体验问题——视时间安排

### 边界来源追溯
边界值可来自以下层级（M5 按优先级从高到低取用）：
1. **VO 注解**：`@NotNull`、`@Size(max=N)`、`@Email` 等——最精确
2. **DM 字段定义**：字段类型+长度+必填——次精确
3. **安全规范/通用规则**：SQL 注入、Unicode——兜底
