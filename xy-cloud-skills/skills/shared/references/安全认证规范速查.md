## 1. 权限控制体系

### 1.1 @PreAuthorize 权限注解
**权限字符串格式**: `'{module}:{entity}:{action}'`

| 段 | 说明 | 常见值 |
|---|---|---|
| `module` | 模块名 | system, infra, eos 等 |
| `entity` | 实体名 | user, dept, role, menu, config 等 |
| `action` | 操作 | create, update, delete, query, export |

```java
@PreAuthorize("@ss.hasPermission('system:user:create')")
@PreAuthorize("@ss.hasPermission('system:user:query')")
@PreAuthorize("@ss.hasPermission('system:dept:update')")
@PreAuthorize("@ss.hasPermission('system:dept:delete')")
@PreAuthorize("@ss.hasPermission('system:permission:assign-role-menu')")
@PreAuthorize("@ss.hasPermission('infra:config:create')")
@PreAuthorize("@ss.hasPermission('infra:job:query')")
@PreAuthorize("@ss.hasPermission('eos:weather:create')")
@PreAuthorize("@ss.hasPermission('eos:alarm-rule:export')")
```
> `@ss` = `SecurityFrameworkServiceImpl` 的 Spring EL Bean 名称。校验不通过抛出 `AccessDeniedException`，返回 403。

### 1.2 @PermitAll 免登录
用于登录、注册、验证码、回调等公开接口。
```java
import javax.annotation.security.PermitAll;
@PermitAll
@PostMapping("/login")
public CommonResult<AuthLoginRespVO> login(@Valid @RequestBody AuthLoginReqVO reqVO) { ... }
```

### 1.3 权限校验服务
- **接口**: `SecurityFrameworkService` — hasPermission, hasAnyPermissions, hasRole, hasAnyRoles, hasScope, hasAnyScopes
- **实现**: `SecurityFrameworkServiceImpl`（Bean 名称 `@ss`）
- **校验方式**: Feign 远程调用 `PermissionApi` + Guava Cache 缓存 1 分钟
```
@PreAuthorize("@ss.hasPermission('system:user:create')")
    → SecurityFrameworkServiceImpl.hasPermission()
        → PermissionApi (Feign 客户端) → 远程校验 → Guava Cache (1 分钟)
```

## 2. 用户上下文

### 2.1 LoginUser 模型
`cn.shutan.platform.framework.security.core.LoginUser`

| 字段 | 类型 | 说明 |
|------|------|------|
| id | Long | 用户 ID |
| userType | Integer | 用户类型 |
| tenantId | Long | 租户 ID |
| scopes | List\<String\> | 权限范围列表 |
| expiresTime | LocalDateTime | 过期时间 |
| info | Map\<String, String\> | 扩展信息（nickname, deptId 等） |
| context | Map\<String, Object\> | 请求生命周期内临时缓存 |

### 2.2 SecurityFrameworkUtils（服务端工具类）
```java
getLoginUser()                      // 获取当前 LoginUser 对象
getLoginUserId()                    // 获取当前用户 ID
getLoginUserNickname()              // 昵称
getLoginUserDeptId()                // 部门 ID
setLoginUser(loginUser, request)    // 设置到 SecurityContext
obtainAuthorization(request, headerName, parameterName)  // 提取 Token
```

### 2.3 WebFrameworkUtils（通用工具类）
```java
getLoginUserId(request)     // 从请求 Attribute 获取登录用户 ID
getLoginUserType(request)   // 从请求 Attribute 获取用户类型
getTenantId(request)        // 从请求 Attribute 获取租户 ID
```

### 2.4 Feign 透传机制
`LoginUserRequestInterceptor` — 微服务间 RPC 调用自动传递用户上下文。
```
服务 A → 从 SecurityContextHolder 获取 LoginUser → 序列化 JSON → Header: login-user
服务 B → 读取 login-user Header → 反序列化 → 设置到 SecurityContextHolder
```
> **约束**: 仅内部可信网络使用。Gateway 层必须过滤外部请求中的 `login-user` Header。

## 3. 数据权限 @DataPermission

### 3.1 注解定义
`cn.shutan.platform.framework.datapermission.core.annotation.DataPermission`

| 属性 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| enable | boolean | true | 是否启用数据权限过滤 |
| includeRules | Class\<\?\>[] | {} | 白名单：仅生效的规则类 |
| excludeRules | Class\<\?\>[] | {} | 黑名单：排除的规则类 |

### 3.2 典型使用场景
```java
// 场景一：关闭数据权限，避免递归查询
@DataPermission(enable = false)
public DeptDataPermissionRespDTO getDeptDataPermission(Long userId) { ... }

// 场景二：缓存方法避免数据权限干扰
@DataPermission(enable = false)
public String getParentDeptPath(Long id) { ... }

// 场景三：编程式忽略
DataPermissionUtils.executeIgnore(() -> {
    List<UserDO> allUsers = userMapper.selectList(...);
});
```

### 3.3 核心机制
| 组件 | 类名 | 职责 |
|------|------|------|
| AOP 拦截器 | `DataPermissionAnnotationInterceptor` | 拦截 `@DataPermission` 方法，推入/弹出栈 |
| 上下文持有者 | `DataPermissionContextHolder` | ThreadLocal 栈结构，维护当前线程上下文 |
| MP 处理器 | `DataPermissionRuleHandler` | SQL 执行前拼接数据权限条件 |
| 编程式工具 | `DataPermissionUtils` | 提供 `executeIgnore()` 等静态方法 |

```
方法调用 → AOP 拦截 → 解析注解 → ContextHolder 入栈
    → MP 执行 SQL → RuleHandler 拼接 WHERE 条件
→ 方法返回 → ContextHolder 出栈
```

### 3.4 部门数据权限规则
`DeptDataPermissionRule` — 自动拼接 SQL WHERE 条件。

| 用户权限设置 | SQL 拼接效果 |
|-------------|-------------|
| 全部可见 | 不拼接额外条件 |
| 部分部门可见 | `dept_id IN (可见部门 ID 列表)` |
| 仅自己可见 | `user_id = 当前用户 ID` |

自定义扩展：
```java
@Bean
public DeptDataPermissionRuleCustomizer customizer() {
    return rule -> {
        rule.addDeptColumn("sys_user", "dept_id");
        rule.addUserColumn("sys_order", "creator_id");
    };
}
```

## 4. 多租户

### 4.1 @TenantIgnore 注解
忽略租户过滤，用于跨租户操作。
```java
@TenantIgnore
private AdminUserDO validateUserForCreateOrUpdate(String username) { ... }

@TenantIgnore
public PageResult<TenantDO> getTenantPage(TenantPageReqVO pageReqVO) { ... }
```

### 4.2 TenantUtils 工具类
```java
TenantUtils.execute(tenantId, () -> { orderService.listOrders(); });              // 指定租户
OrderDO order = TenantUtils.execute(tenantId, () -> orderService.getOrder(orderId)); // 带返回值
TenantUtils.executeIgnore(() -> { List<UserDO> allUsers = userMapper.selectList(...); }); // 跨租户
```

### 4.3 租户过滤机制
| 组件 | 类名 | 职责 |
|------|------|------|
| 数据库拦截器 | `TenantDatabaseInterceptor` | SQL 自动拼接 `tenant_id = ?` |
| 上下文持有者 | `TenantContextHolder` | TransmittableThreadLocal，支持线程池传递 |
| Web 过滤器 | `TenantContextWebFilter` | 从 Header `tenant-id` 提取租户 ID |
| 安全过滤器 | `TenantSecurityWebFilter` | 校验租户合法性，防止越权访问禁用/过期租户 |

```
HTTP 请求 → TenantContextWebFilter 提取 tenant-id → ContextHolder
    → TenantSecurityWebFilter 校验合法性（非法则拒绝）
    → 业务 SQL → TenantDatabaseInterceptor 自动拼接 tenant_id = ?
→ 请求结束 → 清理 ContextHolder
```

### 4.4 租户忽略配置
```yaml
platform:
  tenant:
    ignore-tables:
      - sys_config
      - sys_dict_data
      - sys_dict_type
      - infra_job
```

## 5. API 安全防护

### 5.1 @Idempotent 幂等注解
基于 Redis（SET NX），防止接口重复提交。

| 属性 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| timeout | int | 1 | 超时时间 |
| timeUnit | TimeUnit | SECONDS | 时间单位 |
| message | String | "重复请求，请稍后重试" | 重复请求提示 |
| keyResolver | Class | DefaultIdempotentKeyResolver | Key 解析策略 |
| keyArg | String | "" | 自定义表达式参数 |
| deleteKeyWhenException | boolean | true | 业务异常时是否删除 Key |

**Key 解析策略**:

| 策略类 | 适用场景 |
|--------|----------|
| `DefaultIdempotentKeyResolver` | 全局唯一 Key |
| `UserIdempotentKeyResolver` | 同一用户重复提交 |
| `ExpressionIdempotentKeyResolver` | 基于请求参数精细控制 |

```java
@PostMapping("/create")
@Idempotent(timeout = 5, timeUnit = TimeUnit.SECONDS, message = "请勿重复提交")
public CommonResult<Long> create(@Valid @RequestBody XxxCreateReqVO reqVO) { ... }
```

### 5.2 @RateLimiter 限流注解
基于 Redis 计数器（滑动窗口或固定窗口），防止接口被高频调用。

| 属性 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| time | int | 1 | 时间窗口大小 |
| timeUnit | TimeUnit | SECONDS | 时间单位 |
| count | int | 100 | 窗口内最大请求数 |
| message | String | "请求过于频繁" | 被限流提示 |
| keyResolver | Class | DefaultRateLimiterKeyResolver | Key 解析策略 |

**Key 解析策略**:

| 策略类 | 适用场景 |
|--------|----------|
| `DefaultRateLimiterKeyResolver` | 全局限流 |
| `UserRateLimiterKeyResolver` | 按用户限流 |
| `ClientIpRateLimiterKeyResolver` | 按 IP 限流，防爬虫 |
| `ServerNodeRateLimiterKeyResolver` | 按节点限流 |
| `ExpressionRateLimiterKeyResolver` | 灵活限流粒度 |

```java
@GetMapping("/page")
@RateLimiter(time = 1, timeUnit = TimeUnit.MINUTES, count = 1000,
             message = "请求频率过高，请稍后再试",
             keyResolver = UserRateLimiterKeyResolver.class)
public CommonResult<PageResult<XxxDO>> page(XxxPageReqVO reqVO) { ... }
```

## 6. OAuth2 Token 认证

### 6.1 Token 认证流程
```
客户端请求（携带 Token）
    │
    ├─ Gateway 层: TokenAuthenticationFilter (GlobalFilter)
    │   ├─ 优先检查 login-user Header（网关已解析的优化路径）
    │   ├─ 从 Authorization Header 或 token 参数提取 Token
    │   ├─ 调用 OAuth2TokenApi.checkAccessToken(token) (WebClient)
    │   ├─ 本地缓存校验结果（Guava Cache, 1 分钟）
    │   └─ 写入 login-user Header 转发下游
    │
    ├─ Web 层: TokenAuthenticationFilter (OncePerRequestFilter)
    │   ├─ 优先从 login-user Header 解析（Gateway 透传）
    │   ├─ 其次从 Authorization Header 获取 Token
    │   ├─ 调用 OAuth2TokenApi.checkAccessToken(token) (Feign)
    │   └─ 验证通过后写入 SecurityContext
    │
    └─ 下游服务: 从 SecurityContext 获取 LoginUser → 业务处理
```

### 6.2 双 Token 机制
| Token 类型 | 有效期 | 存储位置 | 用途 |
|-----------|--------|----------|------|
| Access Token | 短期（如 2 小时） | Redis + MySQL | 业务接口认证 |
| Refresh Token | 长期（如 30 天） | 仅 MySQL | 刷新 Access Token |

```
checkAccessToken(token)
    → 1. 优先从 Redis 查询 Token 信息
    → 2. Redis 未命中，从 MySQL 查询
    → 3. 验证有效性（是否过期、是否已撤销）
    → 4. 验证通过后缓存到 Redis
    → 5. 返回对应用户信息
```

### 6.3 Gateway 层处理要点
- Reactive 编程模型（WebFlux），适配异步非阻塞架构
- Guava Cache 缓存 Token 校验结果（1 分钟 TTL）
- 通过 `WebClient` 调用 `OAuth2TokenApi` 校验 Token
- 解析后通过 `login-user` Header 转发下游
- **必须**过滤外部请求中的 `login-user` Header，防止伪造

## 7. 安全配置

### 7.1 SecurityProperties
```yaml
platform:
  security:
    token-header: Authorization
    token-parameter: token
    mock-enable: false
    permit-all-urls:
      - /login
      - /register
      - /captcha/**
      - /callback/**
```

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| `platform.security.token-header` | Authorization | HTTP Header 名称 |
| `platform.security.token-parameter` | token | URL 参数名（WebSocket 等场景） |
| `platform.security.mock-enable` | false | Mock 模式开关（开发环境跳过认证） |
| `platform.security.permit-all-urls` | [] | 免登录 URL 列表 |

### 7.2 模块级安全配置
```java
@Bean
public AuthorizeRequestsCustomizer xxModuleCustomizer() {
    return authorizes -> authorizes
        .requestMatchers("/v3/api-docs/**", "/swagger-ui/**", "/webjars/**").permitAll()
        .requestMatchers("/druid/**", "/actuator/**").permitAll()
        .requestMatchers("/system/rpc/**", "/infra/rpc/**").permitAll()
        .requestMatchers("/infra/file/get/**").permitAll();
}
```

### 7.3 访问拒绝处理
`AccessDeniedHandlerImpl` — 已认证但权限不足时返回 403。
```json
{ "code": 403, "msg": "访问权限不足", "data": null }
```
> 未认证（401）由 `AuthenticationEntryPointImpl` 处理。

## 8. 安全架构总览
```
 Gateway 层
  TokenAuthenticationFilter (GlobalFilter) — Token 验证，缓存，透传 login-user
  TenantContextWebFilter — 从 Header 提取 tenant-id
      │
 Web 层
  TenantSecurityWebFilter — 租户合法性校验 + 防越权检查
  TokenAuthenticationFilter (OncePerRequestFilter) — Token 验证，写入 SecurityContext
      │
 AOP 层
  @PreAuthorize  权限校验  |  @DataPermission  数据权限  |  @TenantIgnore  忽略租户
  @Idempotent  幂等控制   |  @RateLimiter  限流控制
      │
 Feign RPC 层
  LoginUserRequestInterceptor — 透传 LoginUser
  TenantRequestInterceptor — 透传 tenant-id
      │
 MyBatis Plus 层
  TenantDatabaseInterceptor — 自动拼接 tenant_id = ?
  DataPermissionRuleHandler — 自动拼接数据权限 WHERE 条件
```

## 9. 代码生成安全注解模板
{module} / {entity} 替换为实际模块名和实体名（如 system:user, infra:config）。
```java
@PreAuthorize("@ss.hasPermission('{module}:{entity}:create')")
@PostMapping("/create")     public CommonResult<Long> create(...) { ... }

@PreAuthorize("@ss.hasPermission('{module}:{entity}:update')")
@PutMapping("/update")      public CommonResult<Boolean> update(...) { ... }

@PreAuthorize("@ss.hasPermission('{module}:{entity}:delete')")
@DeleteMapping("/delete")   public CommonResult<Boolean> delete(...) { ... }

@PreAuthorize("@ss.hasPermission('{module}:{entity}:query')")
@GetMapping("/get")         public CommonResult<?> get(...) { ... }

@PreAuthorize("@ss.hasPermission('{module}:{entity}:query')")
@GetMapping("/page")        public CommonResult<PageResult<?>> page(...) { ... }

@PreAuthorize("@ss.hasPermission('{module}:{entity}:export')")
@GetMapping("/export")      public void export(...) { ... }
```
