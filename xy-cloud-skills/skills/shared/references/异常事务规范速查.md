# xy-cloud 异常事务规范速查

## 1. 错误码体系

### ErrorCode 类
```java
public class ErrorCode {
    private final Integer code;   // 错误码编号
    private final String msg;     // 错误消息
}
```
- 全局错误码: `[0, 999]` → `GlobalErrorCodeConstants`
- 业务错误码: `[1_000_000_000, +∞)` → `ServiceErrorCodeRange`

### 错误码 10 位四段结构
| 段位 | 长度 | 含义 | 示例 |
|------|------|------|------|
| 第一段 | 1 位 | 类型(1=业务异常) | `1` |
| 第二段 | 3 位 | 系统编号 | `002`(system) |
| 第三段 | 3 位 | 模块编号 | `003`(用户) |
| 第四段 | 3 位 | 具体错误 | `000`~`999` |

完整示例: `1_002_003_000` = 业务异常 > system > 用户 > 第0号错误

### 各模块区间
| 模块 | 区间 |
|------|------|
| infra | `[1-001-000-000, 1-002-000-000)` |
| system | `[1-002-000-000, 1-003-000-000)` |
| eos | `2-002-100-000` 段 |

### 全局错误码
| code | 常量 | 说明 |
|------|------|------|
| 0 | SUCCESS | 成功 |
| 400 | BAD_REQUEST | 请求参数不正确 |
| 401 | UNAUTHORIZED | 账号未登录 |
| 403 | FORBIDDEN | 没有该操作权限 |
| 404 | NOT_FOUND | 请求未找到 |
| 405 | METHOD_NOT_ALLOWED | 请求方法不正确 |
| 423 | LOCKED | 请求失败请稍后重试 |
| 429 | TOO_MANY_REQUESTS | 请求过于频繁 |
| 500 | INTERNAL_SERVER_ERROR | 系统异常 |
| 900 | REPEATED_REQUESTS | 重复请求 |
| 901 | DEMO_DENY | 演示模式禁止写操作 |
| 999 | UNKNOWN | 未知错误 |

### ErrorCodeConstants 定义模式
```java
// 使用 interface 常量定义, 数字下划线分隔提升可读性
public interface ErrorCodeConstants {
    // ========== AUTH 模块: 1-002-000-000 段 ==========
    ErrorCode AUTH_LOGIN_BAD_CREDENTIALS = new ErrorCode(1_002_000_000, "登录失败，账号密码不正确");
    ErrorCode AUTH_LOGIN_USER_DISABLED   = new ErrorCode(1_002_000_001, "登录失败，账号被禁用");

    // ========== 用户模块: 1-002-003-000 段 ==========
    ErrorCode USER_NOT_EXISTS   = new ErrorCode(1_002_003_003, "用户不存在");
    ErrorCode USER_PASSWORD_FAILED = new ErrorCode(1_002_003_005, "用户密码校验失败");
}
```

## 2. 异常类

| 类 | 继承 | 语义 | 说明 |
|----|------|------|------|
| `ServiceException` | RuntimeException(final) | 业务异常 | 禁止继承, 通过 ServiceExceptionUtil 抛出 |
| `ServerException` | RuntimeException(final) | 服务端异常 | 同 ServiceException 结构, 语义不同 |

**禁止直接 `new ServiceException()`，必须通过 `ServiceExceptionUtil.exception()` 抛出。**

## 3. ServiceExceptionUtil

```java
import static cn.shutan.platform.framework.common.exception.util.ServiceExceptionUtil.exception;

// 无参数
throw exception(USER_NOT_EXISTS);

// 带参数 (替换 {} 占位符)
throw exception(ROLE_NAME_DUPLICATE, role.getName());

// 参数校验异常
throw invalidParamException("用户ID不能为空");
```

## 4. 全局异常处理 (GlobalExceptionHandler)

`@RestControllerAdvice` 统一处理:

| 异常类型 | 响应 | 日志 |
|----------|------|------|
| 参数校验异常 | 400 + 错误详情 | warn |
| **ServiceException** | **业务错误码** | **warn 仅首行栈** |
| AccessDeniedException | 403 | warn |
| Exception(兜底) | 500 "系统异常" | error + 异步记录 ApiErrorLog |
| 表不存在 | 501 友好提示 | error |

## 5. 事务规范 @Transactional

**约定: 仅在 Service 层使用。**

```java
@Transactional(rollbackFor = Exception.class)
```

| 方法 | 必须加 | 原因 |
|------|--------|------|
| create | 是 | 多表插入 |
| update | 是 | 多表修改 |
| delete | 是 | 关联检查 |
| query | 否 | 只读操作 |

## 6. 参数校验规范

```java
// Service 接口 — 参数标注 @Valid
public interface XxxService {
    Long create(@Valid XxxSaveReqVO reqVO);
}

// Service 实现 — 类级别 @Validated
@Service @Validated
public class XxxServiceImpl implements XxxService {}
```

### 自定义校验注解
| 注解 | 用途 |
|------|------|
| `@InEnum(GenderEnum.class)` | 校验枚举值范围 |
| `@Mobile` | 手机号格式 |
| `@Telephone` | 座机号格式 |

### 编程式校验
```java
ValidationUtils.validate(object, groups...);
```

## 7. CommonResult

```java
public class CommonResult<T> {
    private Integer code;  // 0=成功, 非0=错误码
    private String msg;
    private T data;
}
```

| 方法 | 说明 |
|------|------|
| `CommonResult.success(data)` | 创建成功响应 |
| `CommonResult.error(code, msg)` | 创建错误响应 |
| `result.isSuccess()` | 判断成功 |
| `result.checkError()` | 失败时抛出 ServiceException |
| `result.getCheckedData()` | 检查并获取数据 |
