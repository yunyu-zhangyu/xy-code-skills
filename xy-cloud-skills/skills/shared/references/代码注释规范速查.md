# 代码注释规范速查

> 用于 AI 代码生成时参考 | JDK 8 | Spring Boot 2.7.18

## 1. 类注释模板

| 层 | 格式 | 示例 |
|----|------|------|
| Controller | `{businessName} Controller` | `/** 能源价格 Controller */` |
| Service 接口 | `{businessName} Service 接口` | `/** 能源价格 Service 接口 */` |
| ServiceImpl | `{businessName} Service 实现类` | `/** 能源价格 Service 实现类 */` |
| Mapper | `{businessName} Mapper 接口` | `/** 能源价格 Mapper 接口 */` |
| DO | `{businessName} DO` | `/** 能源价格 DO */` |
| ReqVO / RespVO | `管理后台 - {businessName} {用途} VO` | `/** 管理后台 - 能源价格新增/修改 Request VO */` |
| 测试类 | `{ClassName} 的单元测试类` | `/** EnergyPriceServiceImpl 的单元测试类 */` |

## 2. 必填检查清单

| # | 检查项 | 规则 |
|---|--------|------|
| 1 | 模板占位符替换 | `${businessName}` `${author}` `${date}` 必须替换，不允许保留 |
| 2 | Controller @Operation | 每个公开方法有 `summary`，复杂接口加 `description` |
| 3 | VO @Schema | 每个字段有 `description` + 推荐 `example` |
| 4 | Service 接口 JavaDoc | 方法有 `@param` + `@return` + 异常说明 |
| 5 | QueryWrapper 条件注释 | 每个 `.likeIfPresent()` 等链式调用加 `// 说明` |
| 6 | 魔数注释 | 所有硬编码数值/字符串必须有注释 |
| 7 | TODO 真实性 | 不允许 `// TODO: 待实现` 等空标记 |

## 3. @Operation 命名规范

| 方法 | summary 格式 | 示例 |
|------|-------------|------|
| POST create | `创建{businessName}` | `"创建能源价格"` |
| PUT update | `更新{businessName}` | `"更新能源价格"` |
| DELETE delete | `删除{businessName}` | `"删除能源价格"` |
| GET get | `获得{businessName}` | `"获得能源价格"` |
| GET list | `获得{businessName}列表` | `"获得能源价格列表"` |
| GET page | `获得{businessName}分页` | `"获得能源价格分页"` |
| GET export | `导出{businessName} Excel` | `"导出能源价格 Excel"` |

## 4. 代码审查导向注释（重要）

AI 生成的注释应帮助 reviewer 快速区分"无需审查的样板代码"和"需要逐行核对的业务逻辑"。

### 4.1 注释标记体系

| 标记 | 含义 | 对 reviewer 的价值 |
|------|------|-------------------|
| `// →` | 自动生成的样板代码，无业务风险 | **跳过审查**，节省时间 |
| `// === 【核查点】 ===` | 此处需要 reviewer 按实际业务补充 | **必看**，审查是否存在遗漏 |
| `// 💡` | 设计决策说明 | 可看可不看，消除疑惑 |
| `// ⚠️` | 静默失败风险（不报错但行为错误） | **必看**，xy-cloud 中最隐蔽的坑 |

### 4.2 各层文件 JavaDoc 模板（必须生成）

```java
/**
 * ${businessName} Controller
 *
 * <p>【AI 生成·核查要点】
 * - @PreAuthorize 权限字符串：格式 '{module}:{resource}:{action}'，需确认与后端权限数据一致
 * - @Tag name 将呈现在 Swagger UI 分组标题，确认中文命名无歧义
 * - @Operation summary 会在 Swagger UI 显示为接口名
 * - @Validated 类级别 + @Valid 参数级别：新加方法也要标注 @Valid
 *
 * @author 张三
 * @date 2026-06-01
 */
```

### 4.3 举例

```java
// → 自动映射，字段名一致即可（无业务风险）
${entity}DO entity = BeanUtils.toBean(reqVO, ${entity}DO.class);

// === 【核查点】是否有唯一性校验？如 name 不能重复 ===
// 示例：if (mapper.selectByName(reqVO.getName()) != null) {
//          throw new ServiceException(XXX_NAME_DUPLICATE);
//      }
```

## 5. 测试方法三段式注释

```java
@Test
public void testCreate${entity}_success() {
    // 准备参数（Arrange）
    // 调用（Act）
    // 断言（Assert）
}
```

// BaseDO     → 系统级共享表（字典/菜单/配置等）
// TenantBaseDO → 租户级隔离表（部门/用户/业务数据等）
