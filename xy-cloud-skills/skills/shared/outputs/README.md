# 输出文件索引

> **总线轨** — 微技能间流水线协作通信的共享目录。相同 tag 重新运行时自然覆盖，下游技能始终读取最新版本。
>
> **存档轨** — 每个技能 `m{N}-{name}/outputs/` 下另有专属历史归档，文件名含 `{YYYYMMDD-HHmm}` 分钟级时间戳。

## m1-需求/ (M1 需求澄清)
包含需求规格文件 `req-{tag}-{module}.md`（YAML frontmatter + 核心功能点 + 数据实体定义）。

## m2-数据模型/ (M2 数据模型)
包含数据模型文件 `dm-{tag}-{entity}.md`（表结构、字段定义、DO 继承选择）。

## m3-组件/ (M3 组件查询)
包含组件推荐清单 `cq-{tag}-{module}-components.md`（可复用组件索引 + 参考代码片段）。

## m4-契约/ (M4 生成契约)
包含 YAML 实现契约文件 `{tag}.yaml`（create/modify 操作清单 + 路径锚点 + 验证规则 + 校验断言）。
代码执行 Agent 按此契约在目标项目中逐条执行。

## m5-边界/ (M5 边界分析)
包含测试边界清单 `bd-{tag}-{entity}.md`（6 种边界类型 + P0/P1/P2 优先级）。

## m6-测试任务/ (M6 测试任务)
包含测试任务书 `{tag}.md`（边界概览 + 测试策略 + 约束清单）。

## m7-测试报告/ (M7 测试执行)
包含测试报告 `tr-{tag}-{module}-{date}.md`（执行概要 + 失败详情 + 结论）。
